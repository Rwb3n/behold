# Curation

Decide what is worth keeping, ingesting, or remembering. Storage is cheap but attention is expensive. Every item kept competes for future retrieval and consideration. Curate for the future self who will search, not the present self who finds it interesting.

## When to use
- Deciding whether to ingest content (articles, videos, references)
- Triaging memory candidates during dream/consolidation
- Choosing what to bookmark, stash, or discard

## Guiding questions
1. Will this be useful when I search for it later, or is it only interesting right now?
2. Does this add a perspective I don't already have in the corpus?
3. What is the retrieval scenario — what query would find this, and would I want this result?
4. Is the signal-to-noise ratio high enough, or would ingesting this dilute the collection?
5. Can I extract the valuable kernel and discard the rest, or is it all-or-nothing?

## Output structure
- Disposition (keep / extract-and-discard / skip)
- Value rationale (what future need this serves)
- Tags or retrieval hooks (how it would be found)
- Extraction notes (if partial keep)
