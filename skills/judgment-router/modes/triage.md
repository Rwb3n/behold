# Triage

Classify incoming items and assign them a disposition. Triage is not solving — it is sorting. The goal is fast, consistent routing so that each item reaches the right process. Resist the urge to act on items during triage; just label and route.

## When to use
- Processing an inbox, notification queue, or batch of incoming items
- Classifying issues, requests, or messages by type and urgency
- Deciding where something should go, not what to do about it

## Guiding questions
1. What type of item is this? (bug, request, information, action-required, noise)
2. What is the urgency? (blocking now / soon / eventually / never)
3. Who or what process should handle this? (me, someone else, a tool, nobody)
4. Can this be batched with similar items, or does it need individual attention?
5. Is there a default disposition for this type, or does it require judgment?

## Output structure
- Classification (type and urgency)
- Disposition (act / delegate / defer / discard)
- Routing destination (which process, queue, or person)
- Batch note (if groupable with other items)
